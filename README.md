# TerraTwin Garden Map

A simple Cloudflare Pages project that lets someone enter a city, estimates a USDA-style hardiness zone from recent historical lows, highlights similar climate regions on a flat global map, and recommends outdoor plants from a 1,000+ record plant database.

## Run locally

```bash
npm install
npm run dev
```

## Build and deploy on Cloudflare

```bash
npm run build
```

The project is configured as a Cloudflare Worker with static assets. This is
compatible with the Workers Builds screen in the Cloudflare dashboard.

- Build command: `npm run build`
- Deploy command: `npx wrangler deploy`
- Node version: 24 or current LTS

The Wrangler configuration uploads `./dist` through Workers Static Assets. To
build and deploy locally in one command, run:

```bash
npm run deploy
```

## Data model

- City search uses the Open-Meteo Geocoding API.
- USDA-zone estimation uses Open-Meteo historical daily minimum temperature data.
- Regional climate fit uses NASA POWER climatology for monthly `T2M_MAX`, `T2M_MIN`, and `RH2M`.
- The flat SVG map uses Natural Earth country boundaries. Countries are clickable and shaded using the nearest cached representative NASA POWER climate profile.
- USDA-style zones are calculated from the average of recent annual extreme lows.
- Plant data is generated by `scripts/generate-plants.mjs` into `public/plants.json`, including category, origin countries, plant type, USDA zone range, preferred warm-season high range, and relative-humidity range.
- Country selection filters plants by botanical origin. Annual and biennial crops use seasonal heat and humidity matching; perennials, shrubs, and trees must also match the selected city's winter-hardiness zone.

USDA hardiness zones are only a winter cold-tolerance signal. The UI calls out that soil, heat, humidity, rainfall, pests, and microclimates still matter.
